Frequently Asked Questions

Learn more about the types of vulnerabilities and how SafeSecureAudit can help safeguard your systems.

What is an XSS vulnerability? Cross-Site Scripting (XSS) occurs when an attacker injects malicious scripts into trusted websites. These attacks exploit flaws where user input is not validated or encoded properly.

Types of XSS:
  • Stored XSS
  • Reflected XSS
  • DOM-based XSS
What is an IDOR vulnerability? Insecure Direct Object References (IDOR) occur when an application exposes internal object references, allowing unauthorized access to data.

Example: A URL like `/api/users/:id` where an attacker can modify the `id` parameter to access data of other users.
What is an SSRF vulnerability? Server-Side Request Forgery (SSRF) allows an attacker to send requests to internal services by exploiting a vulnerability in the server‚Äôs request handling.
What is a SQLi vulnerability? SQL Injection (SQLi) allows an attacker to manipulate SQL queries by injecting malicious code into user input, leading to unauthorized access to databases.
What is an RCE vulnerability? Remote Code Execution (RCE) vulnerabilities occur when an attacker can execute arbitrary code on the server, leading to potential system compromise.
